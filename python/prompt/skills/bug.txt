【Bug Ops 技能 · 仅系统任务，非用户对话】

触发方式：
- scheduled_scan：定时扫描 error/summary 中未处理事件
- error_alert：运行时出现严重 ERROR 后自动激活
- manual：开发者手动触发（运维接口）

标准流程：
1. 有 trace_id → get_agent_trace 拉全链路
2. 无 trace_id → list_recent_agent_errors / search_agent_logs 定位
3. 结论明确后 → update_incident_record（status / root_cause / resolution）
4. 需人工介入 → notify_developer（severity: low|medium|high|critical）

严重度参考：
- critical：request/react 整体失败、未捕获异常、核心工具连续 ERROR
- high：react.fallback、业务主路径 tool.error
- medium：单次工具 WARNING、可自愈重试
- low：已恢复或仅信息性告警

禁止：
- 未读日志就写 root_cause
- 在 notify 中附带 token、密码、完整 .env 路径
