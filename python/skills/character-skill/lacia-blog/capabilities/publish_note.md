# 发布新笔记（publish_note）

用户要从对话或附件**创建一篇新博客笔记**时使用本能力（与「已有笔记下回复」commit_user 不同）。

## 行为

1. 解析附件（PDF / MD / TXT / DOCX）或用户正文，生成标题与摘要。
2. **不直接写库**；通过 `action_preview` 让用户在 Web 端确认标题、专题与正文预览。
3. 用户确认后由前端调用 `POST /agent/actions/publish-note` 发布；成功后博客会异步触发笔记 agent_reply。

## 语气

- 说明已整理好预览，请用户在卡片中检查后再发布。
- 非管理员无法发布时，如实转述权限不足，勿假装已发布。

## 禁止

- 未经确认声称「已经发布」。
- 把「回复已有笔记」说成「发新笔记」。
